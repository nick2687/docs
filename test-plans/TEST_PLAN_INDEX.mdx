## Testing Phases

### Phase 1: Smoke Testing (Week 1)
**Focus**: Critical paths only  
**Documents**: Authentication, Inspections (High priority only)  
**Goal**: Verify core functionality works

### Phase 2: Feature Testing (Weeks 2-3)
**Focus**: All functional test cases  
**Documents**: All feature-specific plans  
**Goal**: Complete feature coverage

### Phase 3: Integration Testing (Week 4)
**Focus**: Feature interactions  
**Documents**: Integration test cases across all plans  
**Goal**: Verify features work together

### Phase 4: Performance & Security (Week 5)
**Focus**: Non-functional requirements  
**Documents**: Performance and Security test cases  
**Goal**: Meet performance targets, no security issues

### Phase 5: Cross-Browser Testing (Week 6)
**Focus**: Browser compatibility  
**Documents**: Cross-browser test cases  
**Goal**: Works on all target browsers

### Phase 6: Regression Testing (Week 7)
**Focus**: Re-test all fixed defects  
**Documents**: All plans (regression suite)  
**Goal**: Ready for production

## Quick Reference

### Critical Path Tests (Must Pass Before Release)

1. **User Authentication** - Sign up, login, logout
2. **Inspection Creation** - Complete multi-step flow
3. **Widget Booking** - Complete booking from widget
4. **Service Selection** - Pricing calculation works
5. **Contact Management** - CRUD operations work
6. **Email Sending** - Emails send successfully
7. **Portal Publishing** - Portal accessible and secure
8. **Template Sync** - Templates sync to mobile
9. **Calendar Integration** - Inspections appear on calendar
10. **Settings Persistence** - Settings save correctly

### High-Risk Areas (Extra Attention Required)

1. **Payment Processing** (when implemented)
2. **Data Security** (user data isolation)
3. **Email/SMS Consent** (TCPA/CAN-SPAM compliance)
4. **Portal Security** (token security, access control)
5. **Widget Availability** (prevent double-booking)
6. **Pricing Calculator** (accurate calculations)
7. **Cross-Platform Sync** (data consistency)
8. **File Uploads** (security, validation)
